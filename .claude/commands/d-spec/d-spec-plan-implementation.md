<!-- D-SPEC:START -->
**Guardrails**
- Only plan from approved proposals and specs.
- Keep tasks small, verifiable, and sequenced.
- Use Beads for execution tracking after approval.

**Steps**
1. Read the approved change docs (proposal, tasks, specs, design if present).
2. Break work into ordered steps with clear outcomes.
3. Flag dependencies and parallelizable work.
4. Ask for confirmation before creating any Beads items.

**Reference**
- Beads handoff prereads are in `.d-spec/onboarding/discovery-to-spec.md`.
